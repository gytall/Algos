# Алгоритмы поиска подстрок и решение головоломок

## Описание

Этот проект посвящен реализации эффективных алгоритмов поиска подстрок в тексте и решению головоломок с использованием алгоритмов поиска.

## Реализованные алгоритмы поиска подстрок

### 1. Алгоритм Кнута-Морриса-Пратта (KMP)
- **Сложность времени**: O(n + m), где n - длина текста, m - длина образца
- **Сложность памяти**: O(m)
- **Описание**: Эффективный алгоритм, использующий предобработку образца для создания таблицы частичных совпадений (LPS - Longest Proper Prefix which is also Suffix). Это позволяет избежать повторных сравнений.

**Преимущества:**
- Не возвращается назад по тексту
- Эффективен для длинных текстов
- Поддерживает поиск без учета регистра

### 2. Упрощенный алгоритм Бойера-Мура
- **Сложность времени**: O(n × m) в худшем случае, O(n/m) в лучшем случае
- **Сложность памяти**: O(m)
- **Описание**: Алгоритм, использующий эвристику "плохого символа" для пропуска символов, которые точно не совпадают. Сравнение начинается с конца образца.

**Преимущества:**
- Очень быстр на практике
- Пропускает большие участки текста
- Эффективен для коротких образцов

## Сравнение с стандартным поиском

Оба алгоритма сравниваются со стандартным методом `str.find()` Python для демонстрации эффективности специализированных алгоритмов.

## Реализованные головоломки

### Задача 2: Пятнашки (15-puzzle)
Реализован алгоритм решения головоломки "Пятнашки" с использованием поиска в ширину (BFS).

**Описание задачи:**
- Игровое поле 4×4 с 15 пронумерованными фишками и одной пустой клеткой
- Цель: расположить фишки в правильном порядке
- Разрешено перемещать фишки только в пустую клетку

**Алгоритм:**
- Поиск в ширину (BFS) для нахождения кратчайшего решения
- Генерация всех возможных состояний
- Проверка достижения целевого состояния

**Сложность:**
- Временная: O(b^d), где b - коэффициент ветвления, d - глубина решения
- Пространственная: O(b^d)

## Особенности реализации

### Алгоритм KMP
- Вычисление таблицы LPS (Longest Proper Prefix which is also Suffix)
- Эффективный переход при несовпадении
- Поддержка поиска без учета регистра

### Алгоритм Бойера-Мура
- Построение таблицы сдвигов для "плохого символа"
- Сравнение справа налево
- Оптимизация пропуска символов

### Пятнашки
- Представление состояния как одномерного списка
- Генерация соседних состояний
- Поиск в ширину для нахождения кратчайшего решения
